### 4. Execution Strategy вЂ” Local

#### Fast Path: `TokenRunner.TryRunAsSystem()`

1. Enable `SeDebugPrivilege`, `SeImpersonatePrivilege`, `SeAssignPrimaryTokenPrivilege` on caller's token.
2. Scan processes for a SYSTEM-token holder (priority order: `winlogon в†’ services в†’ lsass в†’ wininit в†’ all`).
3. `OpenProcessToken` в†’ `DuplicateTokenEx` (primary token, `TOKEN_ALL_ACCESS`).
4. `CreateProcessWithTokenW` вЂ” process runs as SYSTEM.

**Interactive detection logic:**
- `!Console.IsInputRedirected && !Console.IsOutputRedirected` в†’ both stdin and stdout are real console handles в†’ child inherits the parent console (no `STARTF_USESTDHANDLES`), supports interactive programs.
- Otherwise в†’ anonymous pipes + background relay threads в†’ suitable for scripting.

#### Fallback Path: `ServiceRunner.RunLocal()`

- Installs a temporary service `RemoteRunSvc_<GUID>` pointing to `<exePath> --service --workdir <tmpDir>`.
- `tmpDir` = `%TEMP%\RemoteRun_<GUID>`.
- Waits up to 10 s for `SERVICE_RUNNING`, then waits for `SERVICE_STOPPED` (configurable timeout).
- Reads and prints `output.txt`.
- Always `SafeDeleteDirectory(tmpDir)` in `finally`.

### 5. Execution Strategy вЂ” Remote (`ServiceRunner.RunRemote()`)

1. Verify `\\machine\admin$` is reachable.
2. Copy `RemoteRun.exe` to `\\machine\admin$\RemoteRun.exe` (overwrites).
3. Create `\\machine\admin$\RemoteRun_<GUID>\` (temp dir visible as `%SystemRoot%\RemoteRun_<GUID>` on target).
4. Write `config.ini` to temp dir.
5. Service binary path on target: `%SystemRoot%\RemoteRun.exe --service --workdir "%SystemRoot%\RemoteRun_<GUID>"`.
6. `OpenSCManager(machine, ...)` в†’ `CreateService` в†’ `StartService` в†’ wait в†’ collect output в†’ stop/delete.
7. Cleanup: `SafeDeleteDirectory(remoteDir)` + `File.Delete(remoteExe)` вЂ” both best-effort.

### 7. Service Lifecycle & Polling

- State polling: 250 ms intervals using `QueryServiceStatus`.
- Max wait for `SERVICE_RUNNING`: hardcoded 10 s.
- Max wait for `SERVICE_STOPPED`: from `Options.TimeoutSeconds` (default 60 s; 0 = `INFINITE`).
- Overflow guard: `min(TimeoutSeconds, int.MaxValue/1000)` before `* 1000` в†’ prevents `uint` overflow.

### 11. Known Constraints & Edge Cases

| # | Observation |
|---|---|
| 1 | **No stdin piping to child** вЂ” `hStdInput = IntPtr.Zero` in all code paths; interactive stdin works only via console inheritance |
| 2 | **No credential passthrough** вЂ” remote relies solely on the current user's Windows network credentials (NTLM/Kerberos via SMB); no `-u`/`-p` flags |
| 3 | **UTF-8 assumption** вЂ” output capturing always decodes as UTF-8; legacy OEM/ANSI codepage programs may produce garbled output |
| 4 | **Service name conflict** вЂ” uses GUID, so collision is astronomically unlikely but not validated before `CreateService` |
| 5 | **Remote cleanup is best-effort** вЂ” if the process crashes mid-cleanup, the exe and temp dir remain on the remote machine |
| 6 | **No UAC manifest in project file** вЂ” UAC manifest must exist at build time or elevation is done via `ShellExecute "runas"` at runtime (currently using runtime approach) |
| 7 | **Output encoding** вЂ” stdout and stderr are merged in `output.txt` (stdout first, then stderr) вЂ” callers cannot distinguish them |
| 8 | **`NoWait` on service path** вЂ” with `-d`, output is never collected (service runs detached); this is by design but undocumented in `PrintUsage()` |
| 9 | **Interactive mode detection** вЂ” `Console.IsInputRedirected || Console.IsOutputRedirected` covers most cases but not all terminal emulators |
| 10 | **No timeout for `StartServiceCtrlDispatcher`** вЂ” if SCM never calls `ServiceMain`, `ServiceWorker.Run` blocks indefinitely |

### 12. Security Posture

- Requires local Administrator в†’ UAC prompted automatically.
- Remote execution requires Administrator on target machine.
- Token duplication copies the SYSTEM token from a well-known stable process; no injection.
- Temp directory is GUID-randomised; no predictable path for symlink attacks (though `%TEMP%` may be user-writable on some configs).
- Temp directory deleted in `finally` block (local); remote cleanup is best-effort.
- No secrets stored on disk; no credential handling.
- Service binary path uses quoted paths for spaces.
